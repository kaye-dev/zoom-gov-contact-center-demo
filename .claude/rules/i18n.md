# i18n（多言語）コーディング規約

このプロジェクトは `app/i18n/dictionaries.ts` に複数ロケールの辞書を持ち、クライアント側で言語を切り替える。
ユーザー向けの表示文字列を追加・変更する際は、以下を**確認なしで**遵守すること。

## 1. 新規文言は全ロケールに翻訳する

- 表示文字列を追加するときは、対応する全ロケール（現在: `ja` / `en` / `zh-Hans` / `zh-Hant` / `ko`）に
  翻訳を追加する。「日本語だけ」「英語だけ」で済ませない。
- 日本語のみで指定された場合も、意味に忠実な自然な翻訳を各言語へ作成して**同時に**追加する。
  ユーザーに「翻訳しますか？」と確認しない。

## 2. すべての表示文字列はロケール追従

- 見出し・区切りラベル・英語サブタイトル（例: "Find information"）など、画面に出る文字列はすべて
  辞書（`Dictionary` 型 + `dictionaries`）に定義し、ロケールに追従させる。
- コンポーネントに文言をハードコードしない。

## 3. 参照は `useI18n()` 経由

- 文言の参照は必ず `useI18n()` の `t` を通して行う（`Header` / `Footer` / `FindInfo` 等の既存パターンに準拠）。

## 4. 型と辞書を同期させる

- `Dictionary` 型に項目を追加したら、全ロケールへ同じ構造で値を追加し、型エラーを残さない。
- ネスト構造（例: `findInfo.call.title`）も全ロケールで一致させる。
